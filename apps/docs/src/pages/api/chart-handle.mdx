# ChartHandle

`ChartHandle`은 생성된 차트를 제어하는 인터페이스입니다. `createChart` 함수가 반환하는 객체입니다.

## 인터페이스

```typescript
interface ChartHandle {
  // 기본 메서드
  updateData: (dataset: Dataset) => void;
  updateConfig: (config: Partial<ChartConfig>) => void;
  render: () => void;
  destroy: () => void;
  getData: () => Dataset;
  getConfig: () => ChartConfig;

  // 뷰포트 제어
  resize: (width: number, height: number) => void;
  resetViewport: () => void;
  setViewport: (viewport: Viewport) => void;
  zoom: (factor: number, centerX?: number, centerY?: number) => void;
  pan: (deltaX: number, deltaY: number) => void;
  getViewport: () => Viewport;
  getState: () => ChartState;

  // 실시간 데이터
  addPoints: (points: DataPoint[], options?: AddPointsOptions) => void;

  // 증분 렌더링 (실시간 최적화)
  addPointsIncremental: (points: DataPoint[], options?: IncrementalAddPointsOptions) => void;
  updateDataIncremental: (nextData: DataPoint[]) => void;
  getIncrementalState: () => IncrementalRenderState;
  pauseIncremental: () => void;
  resumeIncremental: () => void;
  setIncrementalOptions: (options: Partial<IncrementalRenderOptions>) => void;
}
```

## 메서드

### updateData

데이터를 업데이트하고 차트를 다시 렌더링합니다.

```typescript
updateData(dataset: Dataset): void
```

**매개변수:**
- `dataset`: 새로운 데이터셋

**예제:**

```javascript
// 전체 데이터 교체
chart.updateData({
  points: [
    { x: 1, y: 100 },
    { x: 2, y: 150 },
    { x: 3, y: 120 }
  ]
});

// 데이터 포인트 추가
const currentData = chart.getData();
chart.updateData({
  points: [...currentData.points, { x: 4, y: 180 }]
});

// 데이터 필터링
const filteredData = {
  points: currentData.points.filter(p => p.y > 100)
};
chart.updateData(filteredData);
```

**참고:**
- 원본 데이터는 변경되지 않습니다 (불변성)
- 업데이트 후 자동으로 `render()`가 호출됩니다

### updateConfig

차트 설정을 업데이트하고 다시 렌더링합니다.

```typescript
updateConfig(config: Partial<ChartConfig>): void
```

**매개변수:**
- `config`: 변경할 설정 옵션 (부분 업데이트 가능)

**예제:**

```javascript
// 제목만 변경
chart.updateConfig({ title: 'New Title' });

// 여러 옵션 동시 변경
chart.updateConfig({
  colors: ['#ff0000', '#00ff00'],
  lineWidth: 3,
  showGrid: false
});

// 차트 타입 변경
chart.updateConfig({ type: 'bar' });
```

**참고:**
- 부분 업데이트를 지원합니다
- 지정하지 않은 옵션은 기존 값 유지
- 업데이트 후 자동으로 `render()`가 호출됩니다

### render

차트를 수동으로 다시 렌더링합니다.

```typescript
render(): void
```

**예제:**

```javascript
// 일반적으로 자동 렌더링되므로 직접 호출 불필요
chart.render();

// 특별한 경우에만 사용 (예: 컨테이너 크기 변경 후)
window.addEventListener('resize', () => {
  chart.render();
});
```

**참고:**
- `updateData()`와 `updateConfig()`는 자동으로 렌더링합니다
- 수동 렌더링이 필요한 경우는 드뭅니다

### destroy

차트를 제거하고 리소스를 정리합니다.

```typescript
destroy(): void
```

**예제:**

```javascript
// 차트 제거
chart.destroy();

// React/Vue에서 컴포넌트 언마운트 시
useEffect(() => {
  const chart = createChart(container, dataset, config);

  return () => {
    chart.destroy();
  };
}, []);
```

**수행 작업:**
- Canvas 요소 제거
- 이벤트 리스너 제거
- ResizeObserver 정리
- 메모리 해제

**참고:**
- destroy 후에는 차트를 사용할 수 없습니다
- 메모리 누수 방지를 위해 반드시 호출하세요

### getData

현재 차트의 데이터를 반환합니다.

```typescript
getData(): Dataset
```

**반환값:**
- 현재 데이터셋의 복사본 (불변)

**예제:**

```javascript
const currentData = chart.getData();
console.log(currentData.points.length);

// 데이터 기반으로 통계 계산
const sum = currentData.points.reduce((acc, p) => acc + p.y, 0);
const avg = sum / currentData.points.length;

// 새 데이터 추가
const newData = {
  points: [...currentData.points, { x: 10, y: 50 }]
};
chart.updateData(newData);
```

**참고:**
- 반환된 데이터를 수정해도 차트에 영향 없음 (불변)
- 업데이트하려면 `updateData()` 사용

### getConfig

현재 차트의 설정을 반환합니다.

```typescript
getConfig(): ChartConfig
```

**반환값:**
- 현재 설정의 복사본 (불변)

**예제:**

```javascript
const currentConfig = chart.getConfig();
console.log(currentConfig.type);
console.log(currentConfig.colors);

// 설정 기반으로 새 설정 생성
const newConfig = {
  ...currentConfig,
  title: 'Updated Title',
  colors: ['#ff0000']
};
chart.updateConfig(newConfig);
```

**참고:**
- 반환된 설정을 수정해도 차트에 영향 없음 (불변)
- 업데이트하려면 `updateConfig()` 사용

## 사용 예제

### 실시간 데이터 업데이트

```javascript
const chart = createChart(container, { points: [] }, config);

setInterval(() => {
  const currentData = chart.getData();
  const newPoint = {
    x: currentData.points.length + 1,
    y: Math.random() * 100
  };

  chart.updateData({
    points: [...currentData.points, newPoint].slice(-50) // 최근 50개만 유지
  });
}, 1000);
```

### 반응형 차트

```javascript
const chart = createChart(container, dataset, config);

// 윈도우 크기 변경 시 차트 크기 조정
const resizeObserver = new ResizeObserver((entries) => {
  const { width, height } = entries[0].contentRect;
  chart.updateConfig({ width, height });
});

resizeObserver.observe(container);

// 정리
chart.destroy();
resizeObserver.disconnect();
```

### 인터랙티브 필터링

```javascript
const chart = createChart(container, dataset, config);

// 필터 버튼 클릭 시
filterButton.addEventListener('click', () => {
  const currentData = chart.getData();
  const filteredData = {
    points: currentData.points.filter(p => p.y > threshold)
  };
  chart.updateData(filteredData);
});
```

### 차트 타입 전환

```javascript
const chart = createChart(container, dataset, { type: 'line' });

toggleButton.addEventListener('click', () => {
  const current = chart.getConfig();
  const newType = current.type === 'line' ? 'bar' : 'line';
  chart.updateConfig({ type: newType });
});
```

### React 통합

```jsx
import { useEffect, useRef } from 'react';
import { createChart } from '@xelstack/chart-core';

function Chart({ data, config }) {
  const containerRef = useRef(null);
  const chartRef = useRef(null);

  useEffect(() => {
    // 차트 생성
    chartRef.current = createChart(
      containerRef.current,
      data,
      config
    );

    // 정리
    return () => {
      chartRef.current?.destroy();
    };
  }, []);

  useEffect(() => {
    // 데이터 업데이트
    chartRef.current?.updateData(data);
  }, [data]);

  useEffect(() => {
    // 설정 업데이트
    chartRef.current?.updateConfig(config);
  }, [config]);

  return <div ref={containerRef} />;
}
```

### Vue 통합

```vue
<template>
  <div ref="chartContainer"></div>
</template>

<script setup>
import { ref, onMounted, onUnmounted, watch } from 'vue';
import { createChart } from '@xelstack/chart-core';

const props = defineProps(['data', 'config']);
const chartContainer = ref(null);
let chart = null;

onMounted(() => {
  chart = createChart(chartContainer.value, props.data, props.config);
});

onUnmounted(() => {
  chart?.destroy();
});

watch(() => props.data, (newData) => {
  chart?.updateData(newData);
});

watch(() => props.config, (newConfig) => {
  chart?.updateConfig(newConfig);
});
</script>
```

## 함수형 패턴

ChartHandle은 함수형 프로그래밍 원칙을 따릅니다:

```javascript
// 1. 불변성 - 원본 데이터 변경 없음
const data1 = { points: [{ x: 1, y: 10 }] };
chart.updateData(data1);
console.log(data1); // 변경 없음

// 2. 순수 함수 - getData/getConfig는 새 객체 반환
const config1 = chart.getConfig();
const config2 = chart.getConfig();
console.log(config1 === config2); // false

// 3. 사이드 이펙트 분리 - render만 DOM 조작
const data = chart.getData(); // 순수 함수
chart.render(); // 사이드 이펙트
```

## 타입 정의

```typescript
import type { ChartHandle, Dataset, ChartConfig } from '@xelstack/chart-core';

// ChartHandle 타입 사용
function updateChart(chart: ChartHandle, newData: Dataset) {
  chart.updateData(newData);
}

// 타입 안전성
const data: Dataset = chart.getData(); // OK
const config: ChartConfig = chart.getConfig(); // OK
```

## 증분 렌더링 (Incremental Rendering)

증분 렌더링은 실시간 차트에서 전체 데이터를 다시 그리지 않고 새 데이터만 렌더링하여 성능을 최적화합니다. 10,000개의 기존 데이터에 10개를 추가할 때 기존 방식은 ~30ms가 걸리지만, 증분 렌더링은 ~2ms 내에 완료됩니다.

### addPointsIncremental

새 데이터 포인트를 증분 렌더링으로 추가합니다. 기존 데이터는 다시 그리지 않습니다.

```typescript
addPointsIncremental(points: DataPoint[], options?: IncrementalAddPointsOptions): void
```

**매개변수:**
- `points`: 추가할 데이터 포인트 배열
- `options`: 추가 옵션 (선택)
  - `autoRender`: 자동 렌더링 여부 (기본값: `true`)
  - `autoScroll`: 뷰포트 자동 스크롤 여부 (기본값: `false`)

**예제:**

```javascript
// 단일 포인트 추가
chart.addPointsIncremental([{ x: Date.now(), y: 42 }]);

// 여러 포인트 배치 추가
chart.addPointsIncremental([
  { x: 1, y: 100 },
  { x: 2, y: 150 },
  { x: 3, y: 120 }
]);

// 자동 스크롤과 함께 추가
chart.addPointsIncremental([newPoint], { autoScroll: true });
```

**참고:**
- 기존 `updateData()` 대비 93% 이상 성능 향상
- 프레임 버퍼링이 활성화된 경우 다음 애니메이션 프레임에 일괄 렌더링
- 60fps를 유지하면서 초당 100개 이상의 데이터 처리 가능

### updateDataIncremental

전체 데이터를 제공하면 현재 데이터와 비교하여 변경된 부분만 렌더링합니다.

```typescript
updateDataIncremental(nextData: DataPoint[]): void
```

**매개변수:**
- `nextData`: 다음 상태의 전체 데이터 배열

**예제:**

```javascript
// 현재 데이터: [1, 2, 3, 4, 5]
// 다음 데이터: [1, 2, 3, 4, 5, 6, 7, 8]
// → 델타 계산 후 [6, 7, 8]만 렌더링

chart.updateDataIncremental([
  { x: 1, y: 10 },
  { x: 2, y: 20 },
  { x: 3, y: 30 },
  { x: 4, y: 40 }, // 새로 추가된 데이터
  { x: 5, y: 50 }  // 새로 추가된 데이터
]);
```

**델타 계산 유형:**
- `append`: 끝에 데이터 추가 (가장 일반적)
- `prepend`: 앞에 데이터 추가
- `replace`: 전체 교체 (변경이 복잡한 경우)
- `none`: 변경 없음

### getIncrementalState

증분 렌더링의 현재 상태를 조회합니다.

```typescript
getIncrementalState(): IncrementalRenderState
```

**반환값:**

```typescript
interface IncrementalRenderState {
  totalPoints: number;      // 총 데이터 포인트 수
  pendingPoints: number;    // 렌더링 대기 중인 포인트 수
  frameCount: number;       // 처리된 프레임 수
  averageFrameTime: number; // 평균 프레임 처리 시간 (ms)
  isPaused: boolean;        // 일시정지 상태
  isActive: boolean;        // 활성화 상태
  isOffscreenValid: boolean; // 오프스크린 캔버스 유효 여부
}
```

**예제:**

```javascript
const state = chart.getIncrementalState();

console.log(`총 포인트: ${state.totalPoints}`);
console.log(`대기 중: ${state.pendingPoints}`);
console.log(`평균 프레임 시간: ${state.averageFrameTime.toFixed(2)}ms`);

// 성능 모니터링
if (state.averageFrameTime > 16) {
  console.warn('프레임 드랍 가능성 있음');
}
```

### pauseIncremental / resumeIncremental

증분 렌더링을 일시정지하거나 재개합니다. 일시정지 중에도 데이터는 버퍼에 쌓입니다.

```typescript
pauseIncremental(): void
resumeIncremental(): void
```

**예제:**

```javascript
// 탭이 비활성화되면 일시정지
document.addEventListener('visibilitychange', () => {
  if (document.hidden) {
    chart.pauseIncremental();
  } else {
    chart.resumeIncremental();
  }
});

// 사용자 상호작용 중 일시정지
chart.pauseIncremental();
// ... 줌/팬 작업 ...
chart.resumeIncremental();
```

**참고:**
- 일시정지 중 들어온 데이터는 재개 시 일괄 처리됩니다
- 메모리 사용량 관리를 위해 `maxPoints` 설정을 권장합니다

### setIncrementalOptions

증분 렌더링 설정을 런타임에 변경합니다.

```typescript
setIncrementalOptions(options: Partial<IncrementalRenderOptions>): void
```

**옵션:**

```typescript
interface IncrementalRenderOptions {
  enabled?: boolean;        // 증분 렌더링 활성화 (기본값: true)
  frameBuffering?: boolean; // 프레임 버퍼링 활성화 (기본값: true)
  maxPoints?: number;       // 최대 포인트 수 (기본값: 무제한)
}
```

**예제:**

```javascript
// 최대 포인트 수 설정 (슬라이딩 윈도우)
chart.setIncrementalOptions({ maxPoints: 1000 });

// 프레임 버퍼링 비활성화 (즉시 렌더링)
chart.setIncrementalOptions({ frameBuffering: false });

// 증분 렌더링 비활성화 (기존 방식으로 전환)
chart.setIncrementalOptions({ enabled: false });
```

## 증분 렌더링 사용 예제

### 기본 실시간 차트 (권장)

```javascript
import { createChart } from '@xelstack/chart-core';

const chart = createChart(container, { points: [] }, {
  type: 'line',
  title: 'Realtime Chart'
});

// 최대 500개 포인트 유지
chart.setIncrementalOptions({ maxPoints: 500 });

// 실시간 데이터 스트림
setInterval(() => {
  chart.addPointsIncremental([{
    x: Date.now(),
    y: Math.random() * 100
  }]);
}, 100);
```

### WebSocket과 통합

```javascript
const ws = new WebSocket('ws://localhost:8080/data');

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);

  // 증분 렌더링으로 새 데이터만 추가
  chart.addPointsIncremental([{
    x: data.timestamp,
    y: data.value
  }]);
};

// 연결 끊김 시 일시정지
ws.onclose = () => {
  chart.pauseIncremental();
};
```

### 성능 모니터링 대시보드

```javascript
function createMonitoredChart(container, config) {
  const chart = createChart(container, { points: [] }, config);

  // 상태 모니터링 루프
  setInterval(() => {
    const state = chart.getIncrementalState();

    updateUI({
      totalPoints: state.totalPoints,
      fps: Math.round(1000 / state.averageFrameTime),
      pending: state.pendingPoints
    });
  }, 1000);

  return chart;
}
```
