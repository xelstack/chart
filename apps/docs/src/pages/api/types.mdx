# Types

XelStack Chart의 모든 TypeScript 타입 정의입니다.

## 핵심 타입

### DataPoint

차트의 개별 데이터 포인트를 나타냅니다.

```typescript
interface DataPoint {
  x: number | Date | string;
  y: number;
  series?: string;
}
```

**속성:**
- `x`: X축 값 (숫자, 날짜, 또는 문자열)
- `y`: Y축 값 (숫자만 가능)
- `series`: 시리즈 이름 (다중 라인 차트용, 선택사항)

**예제:**

```typescript
// 숫자 포인트
const point1: DataPoint = { x: 1, y: 10 };

// 날짜 포인트
const point2: DataPoint = { x: new Date(), y: 20 };

// 문자열 포인트 (카테고리)
const point3: DataPoint = { x: 'January', y: 30 };

// 시리즈 포인트
const point4: DataPoint = { x: 1, y: 10, series: 'Line A' };
```

### Dataset

차트 데이터셋을 나타냅니다.

```typescript
interface Dataset {
  points: DataPoint[];
}
```

**속성:**
- `points`: DataPoint 배열

**예제:**

```typescript
const dataset: Dataset = {
  points: [
    { x: 1, y: 10 },
    { x: 2, y: 20 },
    { x: 3, y: 15 }
  ]
};
```

### ChartConfig

차트 설정을 나타냅니다.

```typescript
interface ChartConfig {
  type: 'line' | 'bar' | 'pie' | 'scatter';
  width?: number;
  height?: number;
  title?: string;
  colors?: string[];
  showGrid?: boolean;
  lineWidth?: number;
  pointRadius?: number;
  barWidth?: number;
  barGap?: number;
  innerRadius?: number;
  showLabels?: boolean;
  showPercentage?: boolean;
}
```

**속성:**

#### 공통 옵션
- `type`: 차트 타입 (필수)
- `width`: 차트 너비 (픽셀)
- `height`: 차트 높이 (픽셀)
- `title`: 차트 제목
- `colors`: 색상 배열
- `showGrid`: 그리드 표시 여부

#### Line Chart 옵션
- `lineWidth`: 선 두께
- `pointRadius`: 포인트 반지름

#### Bar Chart 옵션
- `barWidth`: 막대 너비 (상대값)
- `barGap`: 막대 간격 (상대값)

#### Pie Chart 옵션
- `innerRadius`: 내부 반지름 (도넛 차트)
- `showLabels`: 라벨 표시 여부
- `showPercentage`: 퍼센트 표시 여부

#### Scatter Chart 옵션
- `pointRadius`: 포인트 반지름

**예제:**

```typescript
const config: ChartConfig = {
  type: 'line',
  width: 800,
  height: 400,
  title: 'Sales Chart',
  colors: ['#3366ff', '#ff3366'],
  showGrid: true,
  lineWidth: 2,
  pointRadius: 3
};
```

### ChartHandle

생성된 차트를 제어하는 인터페이스입니다.

```typescript
interface ChartHandle {
  // 기본 메서드
  updateData: (dataset: Dataset) => void;
  updateConfig: (config: Partial<ChartConfig>) => void;
  render: () => void;
  destroy: () => void;
  getData: () => Dataset;
  getConfig: () => ChartConfig;

  // 뷰포트 제어
  resize: (width: number, height: number) => void;
  resetViewport: () => void;
  setViewport: (viewport: Viewport) => void;
  zoom: (factor: number, centerX?: number, centerY?: number) => void;
  pan: (deltaX: number, deltaY: number) => void;
  getViewport: () => Viewport;
  getState: () => ChartState;

  // 실시간 데이터
  addPoints: (points: DataPoint[], options?: AddPointsOptions) => void;

  // 증분 렌더링 (실시간 최적화)
  addPointsIncremental: (points: DataPoint[], options?: IncrementalAddPointsOptions) => void;
  updateDataIncremental: (nextData: DataPoint[]) => void;
  getIncrementalState: () => IncrementalRenderState;
  pauseIncremental: () => void;
  resumeIncremental: () => void;
  setIncrementalOptions: (options: Partial<IncrementalRenderOptions>) => void;
}
```

자세한 내용은 [ChartHandle API](/api/chart-handle)를 참조하세요.

### Viewport

차트의 뷰포트(보이는 영역)를 나타냅니다.

```typescript
interface Viewport {
  xMin: number;
  xMax: number;
  yMin: number;
  yMax: number;
  zoomLevel: number;
}
```

**속성:**
- `xMin`: X축 최소값
- `xMax`: X축 최대값
- `yMin`: Y축 최소값
- `yMax`: Y축 최대값
- `zoomLevel`: 줌 레벨 (1.0 = 100%)

**예제:**

```typescript
const viewport: Viewport = {
  xMin: 0,
  xMax: 100,
  yMin: 0,
  yMax: 50,
  zoomLevel: 1.0
};
```

## 증분 렌더링 타입

실시간 차트 최적화를 위한 증분 렌더링 관련 타입입니다.

### IncrementalRenderOptions

증분 렌더링 설정 옵션입니다.

```typescript
interface IncrementalRenderOptions {
  enabled?: boolean;        // 증분 렌더링 활성화 (기본값: true)
  frameBuffering?: boolean; // 프레임 버퍼링 활성화 (기본값: true)
  maxPoints?: number;       // 최대 포인트 수 (기본값: 무제한)
}
```

**속성:**
- `enabled`: 증분 렌더링 활성화 여부
- `frameBuffering`: requestAnimationFrame 기반 프레임 버퍼링 활성화
- `maxPoints`: 최대 보관 포인트 수 (초과 시 오래된 데이터 자동 제거)

**예제:**

```typescript
// 증분 렌더링 설정
chart.setIncrementalOptions({
  enabled: true,
  frameBuffering: true,
  maxPoints: 1000
});
```

### IncrementalAddPointsOptions

포인트 추가 시 옵션입니다.

```typescript
interface IncrementalAddPointsOptions {
  autoRender?: boolean;  // 자동 렌더링 여부 (기본값: true)
  autoScroll?: boolean;  // 뷰포트 자동 스크롤 여부 (기본값: false)
}
```

**속성:**
- `autoRender`: 포인트 추가 후 자동으로 렌더링할지 여부
- `autoScroll`: 새 데이터 추가 시 뷰포트를 자동으로 스크롤할지 여부

**예제:**

```typescript
// 자동 스크롤과 함께 포인트 추가
chart.addPointsIncremental([newPoint], {
  autoRender: true,
  autoScroll: true
});
```

### IncrementalRenderState

증분 렌더링 상태 정보입니다.

```typescript
interface IncrementalRenderState {
  totalPoints: number;        // 총 데이터 포인트 수
  pendingPoints: number;      // 렌더링 대기 중인 포인트 수
  frameCount: number;         // 처리된 프레임 수
  averageFrameTime: number;   // 평균 프레임 처리 시간 (ms)
  isPaused: boolean;          // 일시정지 상태
  isActive: boolean;          // 활성화 상태
  isOffscreenValid: boolean;  // 오프스크린 캔버스 유효 여부
}
```

**속성:**
- `totalPoints`: 현재 버퍼에 저장된 총 데이터 포인트 수
- `pendingPoints`: 다음 프레임에 렌더링될 대기 중인 포인트 수
- `frameCount`: 지금까지 처리된 총 프레임 수
- `averageFrameTime`: 최근 프레임들의 평균 처리 시간 (밀리초)
- `isPaused`: `pauseIncremental()` 호출로 일시정지 상태인지 여부
- `isActive`: 스케줄러가 활성화되어 렌더링 중인지 여부
- `isOffscreenValid`: 오프스크린 캔버스가 유효한 상태인지 여부

**예제:**

```typescript
// 성능 모니터링
const state = chart.getIncrementalState();

console.log(`총 포인트: ${state.totalPoints}`);
console.log(`대기 중: ${state.pendingPoints}`);
console.log(`FPS: ${Math.round(1000 / state.averageFrameTime)}`);

if (state.averageFrameTime > 16) {
  console.warn('프레임 드랍 가능성');
}
```

### DeltaType

배치 업데이트 시 델타 계산 결과 유형입니다.

```typescript
type DeltaType = 'append' | 'prepend' | 'replace' | 'none';
```

**유형:**
- `append`: 끝에 데이터 추가 (가장 일반적인 실시간 패턴)
- `prepend`: 앞에 데이터 추가
- `replace`: 전체 데이터 교체 (변경이 복잡한 경우)
- `none`: 변경 없음

## 함수형 프로그래밍 타입

### PureFunction

순수 함수 타입입니다.

```typescript
type PureFunction<T, R> = (input: T) => R;
type PureFunction2<T1, T2, R> = (input1: T1, input2: T2) => R;
type PureFunctionN<T extends any[], R> = (...inputs: T) => R;
```

**예제:**

```typescript
// 순수 함수: 항상 같은 입력에 같은 출력
const double: PureFunction<number, number> = (x) => x * 2;

const add: PureFunction2<number, number, number> = (a, b) => a + b;

const sum: PureFunctionN<number[], number> = (...nums) =>
  nums.reduce((acc, n) => acc + n, 0);
```

### SideEffect

사이드 이펙트를 수행하는 함수 타입입니다.

```typescript
type SideEffect<T> = (input: T) => void;
type SideEffectWithReturn<T, R> = (input: T) => R;
type AsyncSideEffect<T> = (input: T) => Promise<void>;
```

**예제:**

```typescript
// Canvas 렌더링 (사이드 이펙트)
const render: SideEffect<CanvasRenderingContext2D> = (ctx) => {
  ctx.fillRect(0, 0, 100, 100);
};

// DOM 조작 후 요소 반환
const createElement: SideEffectWithReturn<string, HTMLElement> = (id) => {
  const el = document.createElement('div');
  el.id = id;
  document.body.appendChild(el);
  return el;
};

// 비동기 데이터 로딩
const fetchData: AsyncSideEffect<string> = async (url) => {
  const response = await fetch(url);
  const data = await response.json();
  console.log(data);
};
```

### Pipe & Compose

함수 합성 타입입니다.

```typescript
type Pipe<T, R> = (...fns: Array<(input: any) => any>) => (input: T) => R;
type Compose<T, R> = (...fns: Array<(input: any) => any>) => (input: T) => R;
```

**예제:**

```typescript
import { pipe, compose } from '@xelstack/chart-core';

// pipe: 왼쪽에서 오른쪽으로 실행
const transform = pipe(
  (x: number) => x * 2,
  (x: number) => x + 1,
  (x: number) => x.toString()
);
console.log(transform(5)); // "11"

// compose: 오른쪽에서 왼쪽으로 실행
const transform2 = compose(
  (x: number) => x.toString(),
  (x: number) => x + 1,
  (x: number) => x * 2
);
console.log(transform2(5)); // "11"
```

### Immutable

불변 데이터 타입입니다.

```typescript
type Immutable<T> = Readonly<T>;
type ImmutableArray<T> = ReadonlyArray<T>;
type DeepImmutable<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? DeepImmutable<T[K]>
    : T[K];
};
```

**예제:**

```typescript
// 얕은 불변
const config: Immutable<ChartConfig> = {
  type: 'line',
  width: 800
};
// config.type = 'bar'; // 에러: readonly

// 불변 배열
const points: ImmutableArray<DataPoint> = [
  { x: 1, y: 10 }
];
// points.push({ x: 2, y: 20 }); // 에러: readonly

// 깊은 불변
const state: DeepImmutable<{ chart: ChartConfig }> = {
  chart: { type: 'line', width: 800 }
};
// state.chart.width = 1000; // 에러: readonly
```

## 유틸리티 타입

### PartialExcept

일부 속성을 필수로 유지하는 Partial 타입입니다.

```typescript
type PartialExcept<T, K extends keyof T> = Partial<Omit<T, K>> & Pick<T, K>;
```

**예제:**

```typescript
// type만 필수, 나머지는 선택
type ConfigWithType = PartialExcept<ChartConfig, 'type'>;

const config: ConfigWithType = {
  type: 'line' // type은 필수
  // width, height 등은 선택
};
```

### ValueOf

객체 값들의 타입을 추출합니다.

```typescript
type ValueOf<T> = T[keyof T];
```

**예제:**

```typescript
const ChartTypes = {
  LINE: 'line',
  BAR: 'bar',
  PIE: 'pie',
  SCATTER: 'scatter'
} as const;

type ChartType = ValueOf<typeof ChartTypes>; // 'line' | 'bar' | 'pie' | 'scatter'
```

## 타입 가드

### isDataPoint

```typescript
function isDataPoint(value: unknown): value is DataPoint {
  return (
    typeof value === 'object' &&
    value !== null &&
    'x' in value &&
    'y' in value &&
    typeof (value as any).y === 'number'
  );
}
```

### isDataset

```typescript
function isDataset(value: unknown): value is Dataset {
  return (
    typeof value === 'object' &&
    value !== null &&
    'points' in value &&
    Array.isArray((value as any).points)
  );
}
```
