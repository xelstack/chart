# Functional Programming Guide

XelStack Chart는 함수형 프로그래밍 패러다임을 기반으로 구축되었습니다. 이 가이드에서는 라이브러리가 사용하는 함수형 프로그래밍 원칙과 패턴을 설명합니다.

## 핵심 원칙

### 1. 순수 함수 (Pure Functions)

순수 함수는 동일한 입력에 대해 항상 동일한 출력을 반환하며, 부수 효과가 없습니다.

```typescript
// ✓ 순수 함수
function calculateViewport(data: Dataset): Viewport {
  return {
    minX: Math.min(...data.points.map(p => p.x)),
    maxX: Math.max(...data.points.map(p => p.x)),
    minY: Math.min(...data.points.map(p => p.y)),
    maxY: Math.max(...data.points.map(p => p.y)),
  };
}

// 항상 같은 입력에 대해 같은 출력
const viewport1 = calculateViewport(dataset); // { minX: 0, maxX: 10, ... }
const viewport2 = calculateViewport(dataset); // { minX: 0, maxX: 10, ... }
console.log(viewport1 === viewport2); // false (새 객체)
console.log(JSON.stringify(viewport1) === JSON.stringify(viewport2)); // true (같은 값)
```

```typescript
// ✗ 순수하지 않은 함수 (부수 효과 있음)
let globalState = 0;

function impureFunction(x: number): number {
  globalState++; // 부수 효과: 전역 상태 변경
  return x + globalState;
}
```

**XelStack Chart의 순수 함수 예제:**

- `calculateViewportFromDataset(dataset)` - 데이터셋에서 뷰포트 계산
- `mergeConfig(base, override)` - 두 설정 객체 병합
- `filterPoints(points, predicate)` - 조건에 맞는 포인트 필터링
- `sortPoints(points, compareFn)` - 포인트 정렬

### 2. 불변성 (Immutability)

데이터를 직접 수정하지 않고, 항상 새로운 복사본을 생성합니다.

```typescript
// ✓ 불변성 유지
const originalData = {
  points: [
    { x: 0, y: 10 },
    { x: 1, y: 20 },
  ],
};

// 새로운 포인트 추가 (원본 유지)
const newData = {
  ...originalData,
  points: [...originalData.points, { x: 2, y: 30 }],
};

console.log(originalData.points.length); // 2 (원본 유지)
console.log(newData.points.length); // 3 (새 데이터)
```

```typescript
// ✗ 불변성 위반 (직접 수정)
originalData.points.push({ x: 2, y: 30 }); // ❌ 원본 변경
```

**XelStack Chart에서의 불변성:**

```typescript
const chart = createChart(container, dataset, config);

// updateData는 원본을 변경하지 않음
const newDataset = {
  points: [...dataset.points, { x: 10, y: 50 }],
};

chart.updateData(newDataset);

console.log(dataset.points.length); // 원본 크기 유지
console.log(newDataset.points.length); // 새 데이터 크기
```

### 3. 함수 합성 (Function Composition)

작은 함수들을 조합하여 복잡한 동작을 만듭니다.

```typescript
// 작은 순수 함수들
const filterPositive = (points: DataPoint[]) =>
  points.filter(p => p.y > 0);

const sortByX = (points: DataPoint[]) =>
  [...points].sort((a, b) => a.x - b.x);

const mapToAbsolute = (points: DataPoint[]) =>
  points.map(p => ({ ...p, y: Math.abs(p.y) }));

// 함수 합성
const processPoints = (points: DataPoint[]) =>
  mapToAbsolute(sortByX(filterPositive(points)));

// 또는 pipe 사용
import { pipe } from '@xelstack/chart-core/utils';

const processPoints2 = pipe(
  filterPositive,
  sortByX,
  mapToAbsolute
);

const result = processPoints2(rawPoints);
```

**XelStack Chart의 합성 예제:**

```typescript
import { pipe } from '@xelstack/chart-core/utils';
import {
  filterPoints,
  sortPoints,
  calculateViewport
} from '@xelstack/chart-core/utils';

// 데이터 처리 파이프라인
const processDataset = pipe(
  (dataset: Dataset) => ({
    ...dataset,
    points: filterPoints(dataset.points, p => p.y > 0)
  }),
  (dataset: Dataset) => ({
    ...dataset,
    points: sortPoints(dataset.points, (a, b) => a.x - b.x)
  }),
  calculateViewport
);

const viewport = processDataset(rawDataset);
```

### 4. 고차 함수 (Higher-Order Functions)

함수를 인자로 받거나 함수를 반환하는 함수입니다.

```typescript
// 함수를 인자로 받는 고차 함수
function createFilter<T>(predicate: (item: T) => boolean) {
  return (items: T[]): T[] => items.filter(predicate);
}

// 사용
const filterEven = createFilter((n: number) => n % 2 === 0);
const evenNumbers = filterEven([1, 2, 3, 4, 5]); // [2, 4]
```

**XelStack Chart의 고차 함수 예제:**

```typescript
// 커링된 함수
import { createTransform } from '@xelstack/chart-core/utils';

// 1단계: 변환 함수 생성
const scaleBy = (factor: number) => (value: number) => value * factor;

// 2단계: 적용
const scale2x = scaleBy(2);
const scaled = points.map(p => ({ ...p, y: scale2x(p.y) }));
```

## 함수형 패턴

### Map, Filter, Reduce

기본적인 배열 변환 패턴입니다.

```typescript
const dataset = {
  points: [
    { x: 0, y: 10 },
    { x: 1, y: -5 },
    { x: 2, y: 20 },
    { x: 3, y: -15 },
  ],
};

// Map: 각 요소 변환
const doubled = dataset.points.map(p => ({
  ...p,
  y: p.y * 2
}));

// Filter: 조건에 맞는 요소만 선택
const positive = dataset.points.filter(p => p.y > 0);

// Reduce: 단일 값으로 축약
const sum = dataset.points.reduce((acc, p) => acc + p.y, 0);
const average = sum / dataset.points.length;
```

### Pipe (파이프라인)

함수들을 연쇄적으로 실행합니다.

```typescript
import { pipe } from '@xelstack/chart-core/utils';

const result = pipe(
  (x: number) => x + 1,
  (x: number) => x * 2,
  (x: number) => x - 3
)(5); // ((5 + 1) * 2) - 3 = 9
```

**실전 예제:**

```typescript
import { pipe } from '@xelstack/chart-core/utils';

const processData = pipe(
  // 1. 이상치 제거
  (data: Dataset) => ({
    ...data,
    points: data.points.filter(p =>
      p.y >= -100 && p.y <= 100
    )
  }),

  // 2. 정렬
  (data: Dataset) => ({
    ...data,
    points: [...data.points].sort((a, b) => a.x - b.x)
  }),

  // 3. 정규화
  (data: Dataset) => {
    const max = Math.max(...data.points.map(p => p.y));
    return {
      ...data,
      points: data.points.map(p => ({
        ...p,
        y: p.y / max
      }))
    };
  }
);

const processed = processData(rawDataset);
```

### Compose (합성)

Pipe의 반대 방향으로 함수를 합성합니다.

```typescript
import { compose } from '@xelstack/chart-core/utils';

const result = compose(
  (x: number) => x - 3,
  (x: number) => x * 2,
  (x: number) => x + 1
)(5); // ((5 + 1) * 2) - 3 = 9 (pipe와 동일한 결과)
```

### Currying (커링)

여러 인자를 받는 함수를 단일 인자를 받는 함수들의 체인으로 변환합니다.

```typescript
// 일반 함수
function add(a: number, b: number): number {
  return a + b;
}

// 커링된 함수
function curriedAdd(a: number) {
  return (b: number) => a + b;
}

// 사용
const add5 = curriedAdd(5);
const result = add5(3); // 8
```

**XelStack Chart의 커링 예제:**

```typescript
import { createScaler } from '@xelstack/chart-core/utils';

// 스케일러 팩토리 (커링)
const scalePoints = (factor: number) =>
  (points: DataPoint[]) =>
    points.map(p => ({ ...p, y: p.y * factor }));

// 특정 스케일러 생성
const scaleBy2 = scalePoints(2);
const scaleBy10 = scalePoints(10);

// 사용
const scaled2x = scaleBy2(dataset.points);
const scaled10x = scaleBy10(dataset.points);
```

## 사이드 이펙트 관리

### 순수 함수 vs 사이드 이펙트

XelStack Chart는 순수 함수와 사이드 이펙트를 명확히 구분합니다.

```typescript
// 순수 함수: utils/에 위치
import { calculateViewport, mergeConfig } from '@xelstack/chart-core/utils';

// 사이드 이펙트: effects/에 위치
import { renderToCanvas, createCanvas } from '@xelstack/chart-core/effects';
```

### 사이드 이펙트 분리

```typescript
// ✓ 좋은 예: 순수 함수로 계산, 사이드 이펙트는 분리
function updateChart(chart: Chart, newData: Dataset) {
  // 1. 순수 함수로 새 상태 계산
  const newViewport = calculateViewport(newData);
  const newConfig = mergeConfig(chart.config, { viewport: newViewport });

  // 2. 사이드 이펙트 실행
  renderToCanvas(chart.canvas, newData, newConfig);
}
```

```typescript
// ✗ 나쁜 예: 계산과 사이드 이펙트 혼합
function badUpdateChart(chart: Chart, newData: Dataset) {
  const viewport = calculateViewport(newData);
  chart.canvas.clear(); // 사이드 이펙트 혼합
  const config = mergeConfig(chart.config, { viewport });
  chart.canvas.draw(newData); // 사이드 이펙트 혼합
}
```

### 사이드 이펙트 최소화

```typescript
// createChart는 사이드 이펙트를 내부에 캡슐화
const chart = createChart(container, dataset, config);

// 사용자 코드는 순수 함수처럼 사용
chart.updateData(newDataset); // 내부에서 사이드 이펙트 처리
chart.render();
```

## 타입 안전성

### 타입 추론

TypeScript의 타입 추론을 활용합니다.

```typescript
const dataset = {
  points: [{ x: 0, y: 10 }],
};

// 타입 자동 추론
const chart = createChart(container, dataset, {
  type: 'line',
});

// chart의 타입이 자동으로 추론됨
chart.updateData(newDataset); // ✓ 타입 안전
```

### 제네릭

재사용 가능한 함수형 유틸리티를 만듭니다.

```typescript
// 제네릭 함수
function map<T, U>(
  items: T[],
  fn: (item: T) => U
): U[] {
  return items.map(fn);
}

// 타입 안전한 사용
const numbers = [1, 2, 3];
const strings = map(numbers, n => n.toString()); // string[]
```

### 타입 가드

런타임에 타입을 안전하게 확인합니다.

```typescript
function isLineChart(config: ChartConfig): config is LineChartConfig {
  return config.type === 'line';
}

// 사용
if (isLineChart(config)) {
  // config의 타입이 LineChartConfig로 좁혀짐
  console.log(config.lineWidth);
}
```

## 실전 예제

### 데이터 변환 파이프라인

```typescript
import { pipe } from '@xelstack/chart-core/utils';

// 복잡한 데이터 처리를 순수 함수들로 구성
const prepareChartData = pipe(
  // 1. 원본 데이터 정제
  (raw: RawData[]) => raw.filter(d => d.value !== null),

  // 2. 차트 포인트로 변환
  (raw: RawData[]) => raw.map(d => ({
    x: new Date(d.timestamp).getTime(),
    y: d.value,
  })),

  // 3. 시간순 정렬
  (points: DataPoint[]) => [...points].sort((a, b) => a.x - b.x),

  // 4. 이상치 제거
  (points: DataPoint[]) => {
    const mean = points.reduce((sum, p) => sum + p.y, 0) / points.length;
    const stdDev = Math.sqrt(
      points.reduce((sum, p) => sum + Math.pow(p.y - mean, 2), 0) / points.length
    );
    return points.filter(p => Math.abs(p.y - mean) <= 3 * stdDev);
  },

  // 5. 데이터셋 생성
  (points: DataPoint[]) => ({ points })
);

// 사용
const chartData = prepareChartData(rawApiData);
const chart = createChart(container, chartData, config);
```

## 성능 최적화

### 메모이제이션

계산 결과를 캐싱하여 재사용합니다.

```typescript
// 간단한 메모이제이션
function memoize<T, R>(fn: (arg: T) => R): (arg: T) => R {
  const cache = new Map<T, R>();

  return (arg: T) => {
    if (cache.has(arg)) {
      return cache.get(arg)!;
    }
    const result = fn(arg);
    cache.set(arg, result);
    return result;
  };
}

// 사용
const expensiveCalculation = memoize((data: Dataset) => {
  // 복잡한 계산...
  return calculateViewport(data);
});
```

### 지연 평가 (Lazy Evaluation)

필요할 때만 계산을 수행합니다.

```typescript
// 지연 평가 함수
function lazy<T>(fn: () => T) {
  let cached: T | undefined;
  let computed = false;

  return () => {
    if (!computed) {
      cached = fn();
      computed = true;
    }
    return cached!;
  };
}

// 사용
const getViewport = lazy(() => calculateViewport(largeDataset));

// 실제 필요할 때만 계산
if (needsViewport) {
  const viewport = getViewport(); // 이 시점에 계산
}
```

## 함수형 프로그래밍의 장점

### 1. 테스트 용이성

순수 함수는 테스트하기 쉽습니다.

```typescript
// 순수 함수 테스트
describe('calculateViewport', () => {
  it('should calculate correct viewport', () => {
    const dataset = {
      points: [
        { x: 0, y: 10 },
        { x: 5, y: 20 },
      ],
    };

    const viewport = calculateViewport(dataset);

    expect(viewport).toEqual({
      minX: 0,
      maxX: 5,
      minY: 10,
      maxY: 20,
    });
  });
});
```

### 2. 예측 가능성

동일한 입력은 항상 동일한 출력을 보장합니다.

```typescript
const result1 = mergeConfig(base, override);
const result2 = mergeConfig(base, override);
// result1과 result2는 항상 같은 값
```

### 3. 재사용성

작은 순수 함수들은 다양한 맥락에서 재사용할 수 있습니다.

```typescript
// 재사용 가능한 함수
const sortByX = (points: DataPoint[]) =>
  [...points].sort((a, b) => a.x - b.x);

// 여러 곳에서 사용
const sorted1 = sortByX(dataset1.points);
const sorted2 = sortByX(dataset2.points);
```

### 4. 병렬 처리

부수 효과가 없으므로 안전하게 병렬 처리할 수 있습니다.

```typescript
// 여러 데이터셋을 병렬로 처리
const results = await Promise.all(
  datasets.map(async (dataset) => {
    return calculateViewport(dataset);
  })
);
```
