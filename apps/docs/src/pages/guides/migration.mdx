# 마이그레이션 가이드

다른 차트 라이브러리에서 XelStack Chart로 쉽게 마이그레이션할 수 있습니다.

## Chart.js에서 마이그레이션

### 기본 변환

**Chart.js:**
```javascript
import Chart from 'chart.js/auto';

const chart = new Chart(ctx, {
  type: 'line',
  data: {
    labels: ['Jan', 'Feb', 'Mar'],
    datasets: [{
      label: 'Sales',
      data: [10, 20, 15],
      borderColor: '#3366ff',
      borderWidth: 2
    }]
  },
  options: {
    plugins: {
      title: {
        display: true,
        text: 'Sales Chart'
      }
    },
    scales: {
      y: {
        beginAtZero: true
      }
    }
  }
});
```

**XelStack Chart:**
```javascript
import { createChart } from '@xelstack/chart-core';

const chart = createChart(container, {
  points: [
    { x: 'Jan', y: 10, series: 'Sales' },
    { x: 'Feb', y: 20, series: 'Sales' },
    { x: 'Mar', y: 15, series: 'Sales' }
  ]
}, {
  type: 'line',
  title: 'Sales Chart',
  colors: ['#3366ff'],
  lineWidth: 2,
  showGrid: true
});
```

### 주요 차이점

| Chart.js | XelStack Chart | 설명 |
|----------|----------------|------|
| `new Chart(ctx, config)` | `createChart(container, data, config)` | 함수형 API |
| `data.labels` + `data.datasets` | `data.points` | 단순화된 데이터 구조 |
| `options.plugins.title` | `config.title` | 플랫한 설정 구조 |
| `chart.update()` | `chart.updateData()` / `chart.updateConfig()` | 명확한 메서드 분리 |
| `chart.destroy()` | `chart.destroy()` | 동일 |

## Recharts에서 마이그레이션

### React 컴포넌트 변환

**Recharts:**
```jsx
import { LineChart, Line, XAxis, YAxis, CartesianGrid, Tooltip, Legend } from 'recharts';

function Chart({ data }) {
  return (
    <LineChart width={800} height={400} data={data}>
      <CartesianGrid strokeDasharray="3 3" />
      <XAxis dataKey="name" />
      <YAxis />
      <Tooltip />
      <Legend />
      <Line type="monotone" dataKey="value" stroke="#8884d8" />
    </LineChart>
  );
}
```

**XelStack Chart:**
```jsx
import { useEffect, useRef } from 'react';
import { createChart } from '@xelstack/chart-core';

function Chart({ data }) {
  const containerRef = useRef(null);
  const chartRef = useRef(null);

  useEffect(() => {
    chartRef.current = createChart(
      containerRef.current,
      {
        points: data.map(d => ({ x: d.name, y: d.value }))
      },
      {
        type: 'line',
        width: 800,
        height: 400,
        colors: ['#8884d8'],
        showGrid: true
      }
    );

    return () => chartRef.current?.destroy();
  }, []);

  useEffect(() => {
    if (chartRef.current) {
      chartRef.current.updateData({
        points: data.map(d => ({ x: d.name, y: d.value }))
      });
    }
  }, [data]);

  return <div ref={containerRef} />;
}
```

### 주요 차이점

| Recharts | XelStack Chart | 설명 |
|----------|----------------|------|
| 선언적 컴포넌트 | 명령형 API | 더 직접적인 제어 |
| `<Line dataKey="value" />` | `points: [{ y: value }]` | 데이터 중심 접근 |
| 자동 반응형 | 수동 크기 관리 | 더 세밀한 제어 |
| `<Tooltip />` | 직접 구현 필요 | 커스텀 가능 |

## D3.js에서 마이그레이션

### 저수준 제어 → 고수준 API

**D3.js:**
```javascript
const svg = d3.select('#chart')
  .append('svg')
  .attr('width', 800)
  .attr('height', 400);

const xScale = d3.scaleLinear()
  .domain([0, d3.max(data, d => d.x)])
  .range([0, 800]);

const yScale = d3.scaleLinear()
  .domain([0, d3.max(data, d => d.y)])
  .range([400, 0]);

const line = d3.line()
  .x(d => xScale(d.x))
  .y(d => yScale(d.y));

svg.append('path')
  .datum(data)
  .attr('d', line)
  .attr('stroke', '#3366ff')
  .attr('fill', 'none');
```

**XelStack Chart:**
```javascript
const chart = createChart(container, { points: data }, {
  type: 'line',
  width: 800,
  height: 400,
  colors: ['#3366ff']
});
```

### 언제 D3를 계속 사용해야 하나?

XelStack Chart를 사용하세요:
- ✅ 표준 차트 타입 (line, bar, pie, scatter)
- ✅ 빠른 프로토타이핑
- ✅ 성능 중심 애플리케이션
- ✅ 함수형 프로그래밍 선호

D3를 계속 사용하세요:
- ❌ 복잡한 커스텀 시각화
- ❌ 네트워크 그래프, 트리 다이어그램
- ❌ 지도, 지리 시각화
- ❌ 완전한 저수준 제어 필요

## 일반적인 마이그레이션 패턴

### 1. 데이터 변환 함수 작성

```javascript
// Chart.js 데이터 → XelStack 데이터
function convertFromChartJS(chartJSData) {
  const points = [];

  chartJSData.datasets.forEach(dataset => {
    dataset.data.forEach((value, index) => {
      points.push({
        x: chartJSData.labels[index],
        y: value,
        series: dataset.label
      });
    });
  });

  return { points };
}

// 사용
const xelstackData = convertFromChartJS(chartJSData);
const chart = createChart(container, xelstackData, config);
```

### 2. 설정 매핑 함수

```javascript
// Chart.js 옵션 → XelStack 설정
function mapChartJSOptions(chartJSOptions) {
  return {
    title: chartJSOptions.plugins?.title?.text,
    showGrid: chartJSOptions.scales?.x?.grid?.display !== false,
    colors: chartJSOptions.datasets?.map(d => d.borderColor),
    lineWidth: chartJSOptions.datasets?.[0]?.borderWidth
  };
}
```

## 마이그레이션 도우미 도구

### 자동 변환 스크립트

```javascript
// chartjs-to-xelstack.js
export function convertChartJSToXelStack(chartJSConfig) {
  const { type, data, options } = chartJSConfig;

  // 데이터 변환
  const points = [];
  data.datasets.forEach(dataset => {
    data.labels.forEach((label, i) => {
      points.push({
        x: label,
        y: dataset.data[i],
        series: dataset.label
      });
    });
  });

  // 설정 변환
  const config = {
    type: type,
    title: options?.plugins?.title?.text,
    colors: data.datasets.map(d => d.borderColor || d.backgroundColor),
    showGrid: options?.scales?.x?.grid?.display !== false,
    lineWidth: data.datasets[0]?.borderWidth,
    barWidth: data.datasets[0]?.barThickness,
  };

  return { dataset: { points }, config };
}

// 사용
const { dataset, config } = convertChartJSToXelStack(oldChartConfig);
const chart = createChart(container, dataset, config);
```

## 자주 묻는 질문

### Q: 애니메이션은 지원하나요?

A: 현재 버전에서는 애니메이션을 지원하지 않습니다. 성능과 단순성에 초점을 맞추고 있습니다.

### Q: 플러그인 시스템이 있나요?

A: 함수형 프로그래밍 패러다임을 사용하여 기능을 확장할 수 있습니다. 순수 함수를 조합하여 커스텀 동작을 구현하세요.

### Q: TypeScript 지원은?

A: 네, 완전한 TypeScript 지원을 제공합니다. 모든 타입 정의가 포함되어 있습니다.

### Q: SSR (서버 사이드 렌더링) 지원은?

A: Canvas API를 사용하므로 클라이언트 사이드에서만 렌더링됩니다. SSR 환경에서는 동적 import를 사용하세요.

```jsx
// Next.js 예제
const Chart = dynamic(() => import('./Chart'), { ssr: false });
```

### Q: 번들 크기는 얼마나 되나요?

A: 약 50-60KB (gzip 압축 시 ~15KB)로 매우 가볍습니다.

### Q: 이전 라이브러리와 동시에 사용 가능한가요?

A: 네, 점진적 마이그레이션이 가능합니다. 동일한 페이지에서 다른 차트 라이브러리와 함께 사용할 수 있습니다.

## 마이그레이션 예제

### Chart.js 전체 프로젝트 마이그레이션

**Before:**
```javascript
// charts/SalesChart.js
import Chart from 'chart.js/auto';

export function createSalesChart(data) {
  const ctx = document.getElementById('sales-chart').getContext('2d');

  return new Chart(ctx, {
    type: 'line',
    data: {
      labels: data.map(d => d.month),
      datasets: [{
        label: 'Sales',
        data: data.map(d => d.value),
        borderColor: '#3366ff',
        borderWidth: 2
      }]
    },
    options: {
      responsive: true,
      plugins: {
        title: {
          display: true,
          text: 'Monthly Sales'
        }
      }
    }
  });
}
```

**After:**
```javascript
// charts/SalesChart.js
import { createChart } from '@xelstack/chart-core';

export function createSalesChart(data) {
  const container = document.getElementById('sales-chart');

  return createChart(
    container,
    {
      points: data.map(d => ({
        x: d.month,
        y: d.value,
        series: 'Sales'
      }))
    },
    {
      type: 'line',
      title: 'Monthly Sales',
      colors: ['#3366ff'],
      lineWidth: 2
    }
  );
}
```
