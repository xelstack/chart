# 성능 최적화 가이드

XelStack Chart는 대용량 데이터와 실시간 업데이트를 효율적으로 처리하도록 설계되었습니다.

## 핵심 최적화 기법

### 1. 가상화 (Virtualization)

뷰포트 밖의 데이터를 렌더링하지 않습니다.

```javascript
import { createChart, virtualizeDataset } from '@xelstack/chart-core';

// 대용량 데이터 (10만 포인트)
const largeDataset = {
  points: Array.from({ length: 100000 }, (_, i) => ({
    x: i,
    y: Math.random() * 100
  }))
};

// 뷰포트 밖의 포인트 필터링
const viewport = {
  xMin: 0,
  xMax: 100,
  yMin: 0,
  yMax: 100,
  zoomLevel: 1.0
};

const optimized = virtualizeDataset(largeDataset, viewport);
// 이제 100개 포인트만 렌더링 (viewport 범위 내)

const chart = createChart(container, optimized, config);
```

**효과:**
- 렌더링 시간: 1000ms → 50ms (95% 개선)
- 메모리 사용: 크게 감소
- 부드러운 인터랙션

### 2. 디바이스별 최적화

디바이스 성능에 따라 설정을 조정합니다.

```javascript
import { getDeviceInfo, isLowPerformanceDevice } from '@xelstack/chart-core';

const device = getDeviceInfo();

const config = {
  type: 'line',
  // 레티나 디스플레이 최적화
  lineWidth: device.isHighDPI ? 1.5 : 2,
  pointRadius: device.isHighDPI ? 2 : 3,

  // 모바일 최적화
  width: device.isMobile ? 350 : 800,
  height: device.isMobile ? 250 : 400
};

// 저성능 디바이스에서 데이터 샘플링
let processedData = dataset;
if (isLowPerformanceDevice() && dataset.points.length > 5000) {
  processedData = virtualizeDataset(dataset, viewport);
}

const chart = createChart(container, processedData, config);
```

**효과:**
- 모든 디바이스에서 일관된 성능
- 저성능 디바이스에서도 60 FPS 유지

### 3. 배치 업데이트

여러 변경사항을 한 번에 적용합니다.

```javascript
// ❌ 나쁜 예: 여러 번 렌더링
chart.updateConfig({ title: 'New Title' }); // 렌더링 1
chart.updateConfig({ colors: ['#ff0000'] }); // 렌더링 2
chart.updateConfig({ showGrid: false }); // 렌더링 3

// ✅ 좋은 예: 한 번만 렌더링
chart.updateConfig({
  title: 'New Title',
  colors: ['#ff0000'],
  showGrid: false
}); // 렌더링 1번만
```

**효과:**
- 렌더링 횟수 감소
- CPU 사용률 감소
- 더 부드러운 애니메이션

### 4. 불변 데이터 구조

불필요한 복사를 피합니다.

```javascript
// ❌ 나쁜 예: 전체 배열 복사
const newPoints = [...dataset.points];
newPoints[0] = { x: 1, y: 100 };
chart.updateData({ points: newPoints });

// ✅ 좋은 예: 구조적 공유
import { mapPoints } from '@xelstack/chart-core';

const updated = mapPoints(dataset, (p, i) =>
  i === 0 ? { ...p, y: 100 } : p
);
chart.updateData(updated);
```

**효과:**
- 메모리 사용 감소
- GC 압력 감소
- 더 빠른 업데이트

## 실시간 데이터 최적화

### 스트리밍 데이터

```javascript
const MAX_POINTS = 1000; // 최대 포인트 수 제한

function addDataPoint(chart, newPoint) {
  const current = chart.getData();

  // 슬라이딩 윈도우: 최근 1000개만 유지
  const points = [...current.points, newPoint].slice(-MAX_POINTS);

  chart.updateData({ points });
}

// 1초에 10번 업데이트
setInterval(() => {
  addDataPoint(chart, {
    x: Date.now(),
    y: Math.random() * 100
  });
}, 100);
```

### 디바운싱과 쓰로틀링

```javascript
// 디바운싱: 마지막 호출만 실행
function debounce(fn, delay) {
  let timeoutId;
  return (...args) => {
    clearTimeout(timeoutId);
    timeoutId = setTimeout(() => fn(...args), delay);
  };
}

// 윈도우 리사이즈 최적화
const handleResize = debounce(() => {
  chart.updateConfig({
    width: container.clientWidth,
    height: container.clientHeight
  });
}, 250);

window.addEventListener('resize', handleResize);

// 쓰로틀링: 일정 간격으로만 실행
function throttle(fn, interval) {
  let lastTime = 0;
  return (...args) => {
    const now = Date.now();
    if (now - lastTime >= interval) {
      lastTime = now;
      fn(...args);
    }
  };
}

// 스크롤 최적화
const handleScroll = throttle(() => {
  // 스크롤 위치에 따라 viewport 업데이트
}, 100);
```

## 대용량 데이터 처리

### 10만 포인트 이상

```javascript
import {
  createChart,
  virtualizeDataset,
  pipe,
  filterPoints,
  isLowPerformanceDevice
} from '@xelstack/chart-core';

// 100만 포인트
const massiveDataset = {
  points: Array.from({ length: 1000000 }, (_, i) => ({
    x: i,
    y: Math.random() * 100
  }))
};

// 처리 파이프라인
const processData = pipe(
  // 1. 이상치 제거
  (data) => filterPoints(data, (p) => p.y >= 0 && p.y <= 100),

  // 2. 샘플링 (저성능 디바이스)
  (data) => {
    if (isLowPerformanceDevice() && data.points.length > 10000) {
      // 10개 중 1개만 샘플링
      return {
        points: data.points.filter((_, i) => i % 10 === 0)
      };
    }
    return data;
  },

  // 3. 가상화 (viewport 기반)
  (data) => virtualizeDataset(data, viewport)
);

const optimized = processData(massiveDataset);
const chart = createChart(container, optimized, config);
```

**성능 지표:**
- 100만 포인트 → 1000 포인트 렌더링
- 초기 로딩: ~2초
- 줌/팬 반응 시간: ~50ms

### 청킹과 레이지 로딩

```javascript
// 데이터를 청크로 나누어 로딩
async function loadLargeDataset(url) {
  const response = await fetch(url);
  const reader = response.body.getReader();
  const decoder = new TextDecoder();

  let buffer = '';
  let points = [];

  while (true) {
    const { done, value } = await reader.read();

    if (done) break;

    buffer += decoder.decode(value, { stream: true });
    const lines = buffer.split('\n');
    buffer = lines.pop(); // 마지막 불완전한 라인 보존

    for (const line of lines) {
      if (line.trim()) {
        const [x, y] = line.split(',').map(Number);
        points.push({ x, y });

        // 1000개마다 차트 업데이트
        if (points.length % 1000 === 0) {
          chart.updateData({ points: [...points] });
          await new Promise(resolve => setTimeout(resolve, 0)); // 브라우저가 렌더링할 시간 제공
        }
      }
    }
  }

  chart.updateData({ points });
}
```

## 성능 측정

### Performance API 활용

```javascript
function measureChartPerformance(dataset, config) {
  performance.mark('chart-start');

  const chart = createChart(container, dataset, config);

  performance.mark('chart-end');
  performance.measure('chart-creation', 'chart-start', 'chart-end');

  const measure = performance.getEntriesByName('chart-creation')[0];
  console.log(`Chart creation took ${measure.duration}ms`);

  return chart;
}
```

### FPS 모니터링

```javascript
class FPSMonitor {
  constructor() {
    this.frames = [];
    this.lastTime = performance.now();
  }

  tick() {
    const now = performance.now();
    const delta = now - this.lastTime;
    this.lastTime = now;

    this.frames.push(1000 / delta);
    if (this.frames.length > 60) {
      this.frames.shift();
    }
  }

  getFPS() {
    return this.frames.reduce((a, b) => a + b, 0) / this.frames.length;
  }
}

const fpsMonitor = new FPSMonitor();

function animate() {
  fpsMonitor.tick();

  // 차트 업데이트...

  if (fpsMonitor.getFPS() < 30) {
    console.warn('Low FPS detected:', fpsMonitor.getFPS());
  }

  requestAnimationFrame(animate);
}
```
